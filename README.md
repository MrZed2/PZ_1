# PZ_1
1. **Что такое система контроля версий? Для чего она нужна и где может быть использована?**
   - Система контроля версий - это инструмент для отслеживания изменений в файлах и координации работы над проектом
   - Нужна для: сохранения истории изменений, отката к предыдущим версиям, совместной работы нескольких разработчиков
   - Используется в: разработке ПО, написании книг, любых проектах с текстовыми файлами

2. **Что такое репозиторий?**
   - Репозиторий - это хранилище проекта, где хранятся все файлы и история их изменений
   - Бывает локальный (на компьютере) и удаленный (на GitHub)

3. **Что такое коммит (commit)? Почему каждое изменение должно сопровождаться осмысленным сообщением?**
   - Коммит - это сохранение изменений с созданием точки возврата, как "снимок" проекта в определенный момент
   - Осмысленное сообщение нужно чтобы: понимать, что менялось в этом коммите, легко находить нужные изменения, другим разработчикам было понятно

4. **Объясните разницу между публичным (Public) и приватным (Private) репозиторием. В каком случае какой лучше использовать?**
   - Публичный: видят все, можно клонировать, форкать (для опенсорса, портфолио)
   - Приватный: только вы и приглашенные участники (для коммерческих проектов, личных наработок)
   - Публичный - для открытых проектов, приватный - для всего остального

5. **Что такое ветка (branch) в Git? Как вы можете определить, в какой ветке вы сейчас находитесь, глядя на интерфейс GitHub?**
   - Ветка - это отдельная линия разработки, независимая от других
   - На GitHub текущая ветка показана в выпадающем списке слева сверху (кнопка с названием ветки, обычно "main")
